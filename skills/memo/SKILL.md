# Memo 快速记录技能
## 核心功能
- 自动语义分类：将内容智能分配到备忘录或想法孵化池
- 自动格式化：待办项自动添加`- [ ]`标记，自动添加时间戳
- 附件自动提取：识别内容中的图片/链接，整理到Assets列表
- 非侵入式写入：只追加内容，不修改文件已有结构和格式
## 分类规则
**写入 `raw/moc/备忘录.md` 的内容：**
- 明确的待办任务、计划、安排
- 项目相关的具体事项
- 需要后续跟进的具体工作
- 调研、报告、开发等明确任务
**写入 `raw/moc/想法孵化池.md` 的内容：**
- 临时灵感、突发想法
- 不成熟的创意、思路
- 开放性的思考、假设
- 需要后续深入讨论的模糊想法
## 写入格式
### 待办类内容（写入备忘录）
```markdown
### [HH:MM:SS] 待办
- [ ] [内容]
Assets:
- [链接1]
- [链接2]
```
### 想法类内容（写入孵化池）
```markdown
### [HH:MM:SS] 想法
[内容]
Assets:
- [链接1]
```
## 使用流程
1. 读取用户输入的内容和附件
2. 语义分类判断写入目标文件
3. 提取内容中的所有URL/路径作为附件
4. 按照格式要求生成待写入的内容块
5. 追加到目标文件的末尾
6. 反馈写入成功，告知文件路径
## 示例

`/memo 下周要调研RF收发隔离新方案`

写入`raw/moc/备忘录.md`末尾：
```markdown
### [21:30:00] 待办
- [ ] 下周要调研RF收发隔离新方案
Assets:
```

### [21:30:00] 想法
突然想到可以用图神经网络优化无源建模流程
Assets:

Assets:

## 注意事项
- 保持文件原有格式不变，只在末尾追加内容
- 保留所有原始内容信息，不删减用户输入
- URL自动识别，包括http/https/本地路径等
- 不需要分类到备忘录的具体章节，直接追加到末尾即可
- 所有待办项默认标记为未完成`- [ ]`

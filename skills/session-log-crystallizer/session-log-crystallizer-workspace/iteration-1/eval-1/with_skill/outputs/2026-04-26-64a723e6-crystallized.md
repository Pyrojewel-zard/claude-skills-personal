## 问题排查

| 问题 | 根因 | 解决方案 |
|------|------|----------|
| tmux 启动后自动退出 | 待排查 | 1. 使用 `tmux -f /dev/null` 测试是否配置文件问题<br>2. 使用 `tmux source-file ~/.tmux.conf` 检查配置语法<br>3. 检查 shell 配置文件语法 |

### 常见原因分析

1. **配置文件错误**：`~/.tmux.conf` 语法错误导致 tmux 启动加载失败
2. **Shell 配置问题**：`~/.bashrc` 或 `~/.zshrc` 中有错误命令
3. **WSL 终端类型问题**：`TERM` 环境变量配置错误

## 学习要点

### 技术技巧

- 使用 `tmux -f /dev/null` 可以跳过配置文件启动 tmux，用于隔离配置问题
- 使用 `bash -n ~/.bashrc` 可以检查 shell 脚本语法而不执行
- WSL 环境下 tmux 对 `TERM` 变量敏感，需要正确设置

### 避坑指南

- Claude Code hook 配置错误会导致工具调用无法执行，需要先修复 hook 路径
- 排查问题时应先确认工具可用，否则只能提供通用建议
